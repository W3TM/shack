Digital/Packet modes

DigiPi

As of 2024-11-13, I have been testing a Raspberry Pi Zero 2W running DigiPi 1.91.

Here is the initial configuration procedure:

  1. Flash the software on a microSD card.
  2. Install the microSD card into the Pi.
  3. Connect the Pi to power via the USB micro port farthest from the microSD card.
  4. When the Pi boots, it will create a DigiPi hotspot.
  1. Open a browser and visit http://10.0.0.5/wifi.php
  2. Enter your wifi credentials and save them.
  3. Reboot the Pi.
  4. After the Pi reboots, open a configuration panel by visiting http://digipi.local/.
  1. Complete parameters, especially Callsign, Winlink Password, APRS passcode, Grid Square, Latitude and Longitude (thanks https://grid.radio). For IC-705 via USB, select USB-Connected Radio for Radio Interface. The default parameters (Rig number 3085; Device file ttyACM0, and Baud rate 115200) work well.
  2. Press the initialize button, and then the Reboot button to save the changes and reboot the Pi.
  3. After the Pi reboots, visit http://digipi.local/.
  4. Click on the Screen item below the list of services.

  1. Tune the rig to an APRS frequency (e.g., 144.390 MHz). When you hear packets, you should see callsigns in the window.
  2. To test Winlink, turn the Pat Winlink Email Client on. The Pat Winlink Email indicator should turn green and so should the Rig Control Daemon.

  1. Click on the Pat Email link. The Pat mailbox opens.
  2. Click on the Ready indicator to the right of your callsign.
  3. A Connect to remote node… window will open.
  4. Select “Telnet” for the transport: protocol. Monitor the console to confirm that the DigiPi connected to the default telnet (internet) Winlink server.

Winlink/VARA

  1. Connect the Evolve III laptop via USB. If using the MacBook Pro, connect the rig, start Parallels and log in to Windows 11.
  2. Turn the IC-705 on ([POWER]).
  3. Select the APRS/Packet Memory Group (04) on the IC-705.
    • Press MEMO on the touchscreen, then GROUP.
    • Rotate the MULTI knob to ‘04: APRS/Packet’.
    • Press the highlighted row ‘04: APRS/Packet’.
  4. Use the MULTI knob to tune to 145.090 MHz (the frequency for the W3SWL-10 Winlink node).
  5. Check whether the VHF Packet Preset Memory is in use
    • [MENU]/PRESET.
    • Use the MULTI knob to select ‘4: VHF Packet’.
    • If ‘4: VHF Packet’ is loaded, press UNLOAD.
  6. Start Winlink on the PC.
  7. Open the VARA FM connector.
Note

Here are the settings for the VARA FM modem:

Under Settings/VARA Setup:

Use defaults, but add W3TM and the W3TM Registration Key under VARA Licenses.

Under Settings/SoundCard:

Device Input: Microphone (n- USB Audio CODEC
Device Output: Speakers (n- USB Audio CODEC

Note: the n above depends on the specific machine.

Under Settings/PTT:

PTT Via: CAT
Brand: Icom
Model: IC-705
Port : COM3 for Windows 11 under Parallels on MacBook Pro
Bauds: 19200
RTS : not selected
DTR : not selected
CI-V Address: A4

  1. Start the connection by clicking on Start in the Vara FM Winlink Session window.
Note

Here are some Winlink VARA FM Session-specific parameters:

Under Settings/Vara FM Setup

Keep the defaults, but make sure to select

Automatically launch Vara FM TNC when session is opened
Show VARA FM TNC screen when it’s launched

The latter is optional, but gives additional feedback about what’s happening.

  1. When finished, close the connection by closing the VARA FM Winlink Session window.

Packet radio

  1. Start with the IC-705 off, especially if you are using the MacBookPro and Parallels.
  2. Connect the Evolve III laptop via USB. If using the MacBook Pro, connect the rig, start Parallels and log in to Windows 11.
  3. Select the APRS/Packet Memory Group on the rig.
  4. Tune to 145.030 MHz (W3YA-1 and W3PHB-1 packet nodes).
  5. Check whether the VHF Packet Preset Memory is in use (Menu/Preset). If it is not, load it.
Note

The VHF Packet preset primarily changes the radio’s CI-V port from the default for the IC-705 (A4) to 94, the value used for the IC-7300. The SoundModem application recognizes the IC-7300 CI-V value, but not the IC-705.

  1. Launch SoundModem on the PC.
  2. Monitor the waterfall and the SoundModem window. After a few minutes, there should be packets appearing in the application window.
Note

Here are settings for the SoundModemn application:

Sound Card
Output device: USB Audio CODEC
Input device: USB Audio CODEC

PTT Port
Select PTT port: CAT

Then under Advanced PTT Settings
Radio: IC7300 COM port: COM3 for Windows 11 on Parallels via MacBook Pro
Baudrate: 19200
Data bits: 8
Parity: None
Stop bits: 1
DTR: Off
RTS: Off

Direwolf APRS

Evolve III laptop

  1. Connect the Evolve III laptop via USB. If using the MacBook Pro, connect the rig, start Parallels and log in to Windows 11.
  2. Select the APRS/Packet Memory Group on the rig.
  3. Tune to 144.390 MHz.
  4. Check whether the VHF Packet Preset Memory is in use (Menu/Preset). If it is unload it.
  5. Open the Direwolf configuration file and check that the settings are appropriate for the IC-705.

MacBook Pro (running Parallels)

Note

Since I run a receive-only I-Gate that transmits received packets to APRS-IS already, I do not typically enable APRS-IS digipeating or beaconing on the IC-705/MacBook Pro set-up from my home office.

This means that the station can usually only hear received RF packets, and does not transmit via RF or forward received packets via the Internet to APRS.IS.

As a result, most of the APRS digipeater parameters in direwolf.conf are commented out.

Also, if I’m using APRSIS32 to view packets, I disable ‘APRS-IS Enabled’ and ‘Beaconing Enabled’ under the Enables menu:

  1. Start Parallels (Windows 11).
  2. Select the APRS/Packet Memory Group (04) on the IC-705.
  3. Tune to 144.390 MHz; FM-D mode.
  4. Check whether the VHF Packet Preset Memory is in use (Menu/Preset). If it is unload it.
  5. Open the Direwolf configuration file and check that the settings are appropriate for the IC-705.

These parameters are the minimum needed for a receive only station (no IGate, no APRS-IS, no beaconing):

ADEVICE USB 0 0
AGWPORT 8000
KISSPORT 8001

To IGate heard packets to APRS-IS, we need the following:

MYCALL W3TM-<SSID>
IGSERVER noam.aprs2.net
IGLOGIN W3TM-<SSID> <APRS-pascode>

To beacon via RF, we need the following:

PTT COM4 RTS
PBEACON delay=1  every=30 overlay=S symbol="dot" lat=40^47.87N long=077^30.95W power=10 height=10 gain=1 comment="State College PA" via=WIDE1-1

To digipeat heard packets, we set the following:

#DIGIPEAT 0 0 ^WIDE[3-7]-[1-7]$|^TEST$ ^WIDE[12]-[12]$
  1. Start Direwolf (This PC\Local Disk(C:)\direwolf-1.7.0-9807304-_i686)

Local VHF/UHF phone operations

  1. Select the Centre Repeaters memory group.
  2. Check whether the VHF Packet Preset Memory is in use (Menu/Preset). If it is unload it.
  3. Verify that the radio is in FM mode, not FM-D mode. In FM-D mode, audio input is accepted only via USB (see below).
  4. Open the audio scope (Menu/Audio) to monitor output audio.
  5. Choose the repeater you wish to use and make a test transmission.
Note

If needed, check the connectors settings (Menu/Set/Connectors/MOD Input):

DATA OFF MOD: MIC, USB
DATA MOD: USB
